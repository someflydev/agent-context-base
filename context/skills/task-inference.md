# Task Inference

Use this skill to turn a normal-language request into a workflow-first context bundle.

## Procedure

1. restate the task in plain language
2. map it to one primary workflow
3. infer stack from repo signals and touched files
4. infer archetype only if project shape matters
5. load one preferred example if implementation shape is still unclear

## Good Triggers

- "add"
- "fix"
- "refactor"
- "deploy"
- "seed"
- "index"
- "bootstrap"

## Avoid

- loading multiple workflows when one clearly dominates
- requiring the user to name the workflow file explicitly
