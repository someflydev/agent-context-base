# Backend Extension Candidates

Purpose: keep additional backend options structurally compatible with the main architecture.

Included candidates:

- Nim with Jester or HappyX
- Zig with Zap or Jetzig
- Scala with Tapir, http4s, ZIO
- Clojure with Kit, next.jdbc, Hiccup
- Kotlin with http4k and Exposed
- Crystal with Kemal and Avram
- OCaml with Dream, Caqti, TyXML
- Dart with Dart Frog

Shared expectation:

- each candidate needs its own stack pack once it becomes active in a real repo
- each active candidate should define common paths, test expectations, smoke expectations, and one preferred endpoint example
