## Validation-First Autonomy

Autonomy is bounded by proof, not by confidence. An agent may continue unattended only when the repo already states what must be true and how that truth will be checked.

If the validation path is unclear, the correct move is to narrow scope or stop, not to improvise a vague “done” state.

Validation may be skipped only by explicit operator instruction. When that happens, record the skip plainly instead of implying the slice is proven.
