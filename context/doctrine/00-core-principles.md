# Core Principles

Purpose: define the durable operating model for agent-oriented repos.

Rules:

- Keep doctrine separate from workflows, stacks, and examples.
- Load the smallest relevant bundle first.
- Prefer canonical examples over invented patterns.
- Keep top-level router files short and delegated.
- Favor predictable, boring file names over clever taxonomy.
- Optimize for human maintainability as much as model performance.

Prevents:

- giant instruction files
- context sprawl
- duplicate or conflicting guidance
