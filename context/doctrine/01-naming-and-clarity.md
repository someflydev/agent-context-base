# Naming And Clarity

Purpose: reduce ambiguity in file names, docs, and generated artifacts.

Rules:

- Use deterministic descriptive names.
- Keep stack and archetype slugs stable once published.
- In prompt-first repos, keep prompt numbering monotonic.
- Reference files explicitly by name when a task depends on them.
- Prefer one clear term per concept; do not alias casually.

Prevents:

- hidden intent
- duplicate concepts with different names
- routing confusion
