# MEMORY.md Template

Use this as the reference shape for a repo-local `MEMORY.md`.

Keep the file short, current, and operational.

```md
# MEMORY.md

## Current Objective
- State the current task in 1 to 2 sentences.

## Current Context
- Active stack:
- Active archetype:
- Active manifest or profile:
- Prompt batch or deployment posture:

## Active Working Set
- path/to/file

## Files Already Inspected
- path/to/file

## Important Findings
- Fact learned from the repo, code, docs, or validation.

## Decisions Already Made
- Decision:
  Reason:

## Explicitly Not Doing
- Scope guardrail or ruled-out adjacent work.

## Blockers Or Risks
- Risk, failed validation, or missing information.

## Next Steps
- Concrete next action.
- Concrete next action.

## Stop Condition
- Define the next meaningful pause point.

## Last Updated
- YYYY-MM-DD HH:MM local time - short phase note
```

Notes:

- rewrite stale sections instead of appending a long history
- use exact file paths
- prefer current operational facts over narrative
- move durable transition state into a handoff snapshot when a fixed checkpoint matters
