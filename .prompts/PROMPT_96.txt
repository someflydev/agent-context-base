================================================================================
BACKGROUND: WHY AN EXECUTABLE BUDGET MODEL
================================================================================

`context/doctrine/context-complexity-budget.md` contains a complete, well-specified
cost model for context bundles: artifact types with base costs, size cost formula,
modifiers (direct-match discount, cross-domain penalty, etc.), diversity penalty,
bundle formula, hard caps, and six named profiles (tiny through cross-system).

This doctrine is 100% normative today — there is no code that executes it.

The Session Context Briefing added in PROMPT_94 includes a "Complexity Budget" section,
but it only reports a line-count heuristic (lean / moderate / heavy). That is useful
but it is not the same thing as evaluating a declared bundle against the actual cost
model.

The gap: an assistant can declare "I loaded these 7 files for this task" and currently
there is no way to score that bundle, check it against a profile, or print why each file
was included or excluded.

This prompt closes that gap with:
1. A pure Python budget library: `scripts/context_budget.py`
2. A `work.py budget-report` subcommand that scores a declared bundle
3. Tests in `verification/unit/test_context_budget.py`
4. A small update to `context-complexity-budget.md` noting the executable companion

The library is convention-based where it must be (artifact type is inferred from file
path, not from a hidden registry; routing modifiers require declared context) and
deterministic where it can be (scoring formula, profile matching, hard-cap checking).

1.3 Size cost

------------------------------------------------------------------------
1.4 Modifier context
------------------------------------------------------------------------

Modifiers from Section 6.3 that can be applied:

  direct_match_discount:          -1  (artifact explicitly matches declared primary stack/archetype/workflow)
  preferred_example_discount:     -1  (artifact is declared as preferred example)
  direct_manifest_required_discount: -1  (artifact is in required_context)
  cross_domain_penalty:           +2  (artifact introduces concepts outside dominant route)
  secondary_stack_penalty:        +2  (artifact belongs to first supporting stack)
  third_stack_penalty:            +4  (artifact belongs to stack beyond first support)
  extra_archetype_penalty:        +3  (artifact implies second archetype in first pass)
  deployment_without_trigger_penalty: +4  (deployment doctrine without explicit trigger)
  large_example_penalty:          +2  (large_example that is not primary example)

Design a `ModifierContext` dataclass:

  @dataclass
  class ModifierContext:
      primary_stack: str | None = None       # e.g., "python-fastapi"
      primary_archetype: str | None = None   # e.g., "backend-api-service"
      active_workflow: str | None = None     # e.g., "add-api-endpoint"
      required_files: tuple[str, ...] = ()   # declared required context
      preferred_example: str | None = None   # the one preferred example path
      deployment_trigger: bool = False       # whether a deployment trigger was declared
      secondary_stacks: tuple[str, ...] = () # supporting stacks (for penalty calc)
      declared_primary_example: str | None = None  # for large_example_penalty check

All modifier context is optional. When not provided, only base + size cost is computed
(conservative — no discounts applied). This is correct behavior for an undeclared bundle.

Expose: `compute_modifiers(path: str, artifact_type: str, ctx: ModifierContext) -> int`

------------------------------------------------------------------------
1.5 Artifact cost
------------------------------------------------------------------------

  artifact_cost(a) = base_cost(type(a)) + size_cost(a) + compute_modifiers(a, type(a), ctx)

Expose: `ArtifactScore` dataclass:

  @dataclass
  class ArtifactScore:
      path: str
      artifact_type: str
      base_cost: int
      size_cost: int
      modifier_cost: int
      total_cost: int
      modifier_notes: tuple[str, ...]   # human-readable explanation of each modifier applied

------------------------------------------------------------------------
1.6 Bundle cost + diversity penalty
------------------------------------------------------------------------

  diversity_penalty(B) = max(0, distinct_concept_tags(B) - 4)

For concept tags: use a simple path-based tag classifier:
  - paths containing "storage" or "database" → tag: storage
  - paths containing "deploy" or "dokku" → tag: deployment
  - paths containing "workflow" or "smoke" or "test" → tag: verification
  - paths containing "stack" → tag: stack
  - paths containing "rag" or "vector" or "search" → tag: search
  - paths containing "prompt" or "memory" or "session" → tag: session-management
  - all others → tag: general

  bundle_cost(B) = sum(artifact_cost(a) for a in B) + diversity_penalty(B)
                  + ambiguity_penalty (passed in by caller, default 0)
                  + confidence_penalty (passed in by caller, default 0)
                  + change_surface_penalty (passed in by caller, default 0)

Task-level penalty inputs:
  ambiguity_penalty = 2 * ambiguity_level  (level 0-3, default 0)
  confidence_penalty = 0 | 2 | 4 based on confidence (>= 0.85 = 0, 0.70-0.84 = 2, < 0.70 = 4)
  change_surface_penalty = max(0, change_surface_area - 1)  (area 1-4, default 0)

Expose: `BundleScore` dataclass:
  @dataclass
  class BundleScore:
      artifacts: tuple[ArtifactScore, ...]
      subtotal: int                    # sum of artifact costs
      diversity_penalty: int
      ambiguity_penalty: int
      confidence_penalty: int
      change_surface_penalty: int
      total: int
      distinct_concept_tags: int
      stack_count: int                 # inferred from stack_pack artifacts
      archetype_count: int             # inferred from archetype_pack artifacts
      workflow_count: int
      example_count: int
      large_example_count: int

Implement the six profiles from Section 8 as a tuple of frozen dataclasses:

  PROFILES: tuple[BudgetProfile, ...]

Each profile: tiny, small, medium, large, cross-system (also include agent-session if
it exists in the doc — inspect the full file).

  @dataclass(frozen=True)
  class BudgetProfile:
      name: str
      max_points: int
      max_files: int
      max_stacks: int
      max_archetypes: int
      max_workflows: int
      max_examples: int
      max_large_examples: int

Expose:
  `select_profile(bundle_score: BundleScore) -> BudgetProfile | None`
  Returns the tightest profile that the bundle fits. Returns None if the bundle
  exceeds all profiles.

  `check_hard_caps(bundle_score: BundleScore, profile: BudgetProfile) -> list[str]`
  Returns a list of cap violations (empty list if none).

PART 2: work.py budget-report subcommand
================================================================================

Add `budget-report` to `scripts/work.py`.

Command signature:

  python3 scripts/work.py budget-report \
      [--bundle path1 path2 ...] \
      [--bundle-file bundle.json] \
      [--primary-stack <stack>] \
      [--archetype <archetype>] \
      [--workflow <workflow>] \
      [--ambiguity 0-3] \
      [--confidence 0.0-1.0] \
      [--change-surface 1-4] \
      [--json]

Input:
  --bundle path1 path2 ...   explicit file paths (space-separated)
  --bundle-file FILE          path to a JSON file containing {"files": ["path1", ...],
                              "primary_stack": "...", "archetype": "..."}
  (if neither: print usage and exit 1)

Output (default — human-readable markdown):

  Budget Report
  =============
  Bundle: <N files>  Context: <primary_stack | unspecified> / <archetype | unspecified>

  Artifact Scores:
    path/to/file.md        doctrine           base=3  size=0  mods=-1  total=2  [direct match]
    path/to/stack.md       stack_pack         base=4  size=1  mods=+2  total=7  [secondary stack]
    ...

  Bundle Summary:
    Artifact subtotal:     <N>
    Diversity penalty:     <N>  (<K> distinct tags: storage, verification, ...)
    Task-level penalties:  <N>  (ambiguity=<N>, confidence=<N>, change_surface=<N>)
    Total cost:            <N>

  Profile Fit:
    Selected profile:      <name> (max_points=<N>, max_files=<N>)
    Cap violations:        none  |  <violation1>, <violation2>, ...
    Recommendation:        <"fits <profile>" | "exceeds all profiles — split or reduce">

Output (--json):

  {
    "bundle_cost": <int>,
    "artifact_count": <int>,
    "selected_profile": "<name>" | null,
    "cap_violations": [],
    "diversity_penalty": <int>,
    "distinct_concept_tags": <int>,
    "artifacts": [
      {"path": "...", "type": "...", "total_cost": <int>},
      ...
    ]
  }

Add budget-report to `build_parser()`. Use subparser pattern consistent with existing commands.
