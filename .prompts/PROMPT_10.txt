You are performing a major infrastructure pass on the existing repository.

The repository is an "agent-context-base" system designed to help coding assistants (Codex, Claude, Gemini) understand architectures, stacks, archetypes, workflows, and canonical examples for building software systems.

Canonical examples across many languages and frameworks are becoming increasingly important to this repository.

However, they currently lack a systematic verification layer.

Your task is to implement a complete, structured verification and example-testing framework for the repository.

The goal is to ensure that canonical examples, helper scripts, manifests, prompts, and structural conventions remain correct, runnable, and trustworthy.

This system must:

- verify canonical examples across languages
- validate repo integrity
- verify Python maintenance scripts
- compile or run language examples when possible
- run minimal harness tests for API examples
- verify prompt-first repo conventions
- allow incremental addition of new stacks
- provide confidence levels for canonical examples
- integrate well with Docker where appropriate
- remain maintainable and easy to extend

The system must also help assistants prefer verified canonical examples when generating implementations.

The verification layer must be designed to run:

- locally
- in CI
- incrementally
- selectively by stack

The design should avoid requiring every language toolchain to be installed locally.

Instead, canonical examples that require runtimes should use small Docker environments.

The output must be organized into commit-ready batches.

For each batch:

1. Provide a title
2. Explain the purpose
3. List files created or modified
4. Provide the full contents of each file
5. Provide a suggested Tim Pope style commit message starting with a prefix like `[VERIFY_XX]`

Do not skip file contents.
