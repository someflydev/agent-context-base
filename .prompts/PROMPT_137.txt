================================================================================
BACKGROUND: GO AND RUST AUTH PATTERNS
================================================================================

Both Go and Rust have strong type systems that should be used to make auth
context impossible to bypass — not just difficult. In Go, middleware attaches
AuthContext to the Echo Context and handlers receive it via type-asserted
context values. In Rust, Axum's FromRequestParts trait makes auth context an
extractor — handlers that require auth simply declare it as a parameter.

IMPORTANT for Go:
  - Use golang-jwt/jwt (v5 preferred). This is NOT the same as jwt-go.
  - Import path: github.com/golang-jwt/jwt/v5
  - golang-jwt/jwt does NOT have built-in JWKS or OIDC support — that is
    intentional for this canonical example (single-service, symmetric or
    asymmetric key loaded from env/file).
  - Route metadata is a static slice of structs, not a map — iteration
    order matters for middleware lookup.
  - lestrrat-go/jwx is noted in the stack file as the alternative — do not
    use it in the canonical example.

IMPORTANT for Rust:
  - Use the jsonwebtoken crate (latest stable, 9.x).
  - Claims struct must derive Deserialize + Serialize.
  - Decoding errors should map to 401; permission errors to 403.
  - Axum Extension or a custom FromRequestParts extractor for auth context —
    the extractor pattern is preferred because it makes auth a compile-time
    type requirement on handlers that need it.
  - jwt-simple is noted in the stack file as the alternative — do not use
    it in the canonical example.

  auth/middleware.rs:

    Implement AuthContext as an Axum extractor:

      impl<S> FromRequestParts<S> for AuthContext
      where S: Send + Sync

    The extractor:
    1. Reads Authorization: Bearer <token> header
    2. Verifies token with jsonwebtoken::decode::<Claims>(token, &key, &validation)
       where validation checks exp, nbf, iss, aud
    3. Returns StatusCode::UNAUTHORIZED on any verification failure
    4. Looks up user from the store (via State<Arc<InMemoryStore>>)
    5. Compares acl_ver; returns UNAUTHORIZED if stale
    6. Verifies tenant membership; returns FORBIDDEN if inactive
    7. Returns AuthContext on success

    Note: Axum's FromRequestParts requires access to the app state.
    Use the extension-based pattern or pass the store via Extension.
    Read context/stacks/rust-axum-jsonwebtoken-rbac.md for the
    recommended approach.

  RBAC enforcement:
    Permission guards as Axum extractors or middleware:

      pub struct RequirePermission<const PERM: &'static str>(pub AuthContext);

      impl<S, const PERM: &'static str> FromRequestParts<S> for RequirePermission<PERM>
