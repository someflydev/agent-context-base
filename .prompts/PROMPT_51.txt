SHARED SPECIFICATION

Every language example must implement the following shared contract. Do not deviate
from the names, values, or attribute strings. The Playwright test suite is
language-agnostic; it tests against the HTML contract, not the language.


DATASET (identical in every language)

Six rows, inline as a constant:

  daily-signups     | team: growth    | status: active   | region: us   | events: 12
  trial-conversions | team: growth    | status: active   | region: us   | events: 7
  api-latency       | team: platform  | status: paused   | region: eu   | events: 5
  checkout-failures | team: growth    | status: active   | region: eu   | events: 9
  queue-depth       | team: platform  | status: active   | region: apac | events: 11
  legacy-import     | team: platform  | status: archived | region: us   | events: 4

FACET_OPTIONS (three dimensions, same values in every language):
  team:   ["growth", "platform"]
  status: ["active", "paused", "archived"]
  region: ["us", "eu", "apac"]

QUICK_EXCLUDES (preconfigured, not derived from data):
  [("status", "archived"), ("status", "paused")]


QUERY STATE MODEL

Each implementation must define an explicit typed structure with six fields:

  team_in    team_out
  status_in  status_out
  region_in  region_out

Each field holds a sorted, deduplicated list/array/sequence of lowercase trimmed strings.
Incoming parameter values must be normalized to this form before any query logic runs.
Multi-value query parameters (e.g., ?status_out=archived&status_out=paused) must be
parsed as lists, not as comma-separated strings.


REQUIRED FUNCTIONS (named idiomatically per language, behavior identical)

  filter_rows(state)
    Returns rows passing all dimension filters.
    For each dimension: if _in is non-empty, value must be in _in.
    If _out is non-empty, value must not be in _out.

  facet_counts(state, dimension)
    Returns a count per option for the include sub-section.
    Applies all other dimensions' filters fully.
    Relaxes same-dimension INCLUDES (does not apply _in for this dimension).
    Still applies same-dimension EXCLUDES.
    Result: map/dict of option → int.

  exclude_impact_counts(state, dimension)
    Returns a count per option for the exclude sub-section.
    Applies all other dimensions' filters fully.
    Does NOT apply any of this dimension's excludes or includes.
    Result: map/dict of option → int (counts rows that each exclude would remove).

  render_filter_panel(state)
    Returns an HTML string (or rendered template / typed HTML value).
    Must produce the complete filter panel fragment with all data-* attributes.
    See RENDERING CONTRACT below.

  render_results_fragment(state)
    Returns an HTML string containing:
    - An OOB result-count badge: id="result-count", hx-swap-oob="true",
      data-role="result-count", data-result-count="{N}"
    - A results section: id="report-results", data-query-fingerprint="...",
      data-result-count="{N}"

  fingerprint(state)
    Returns a deterministic string representation of the full filter state
    for use in data-query-fingerprint attributes. Example format:
    "team_in=growth|team_out=|status_in=|status_out=archived|region_in=|region_out="


ENDPOINTS (identical paths and behavior in every language)

  GET /ui/reports
    Full dashboard page. Accepts all filter params. Returns full HTML page
    containing the filter panel and results section. HTMX form target.

  GET /ui/reports/results
    HTMX partial. Returns render_results_fragment(state) as HTML.
    Triggered by form change/submit in the filter panel.

  GET /ui/reports/filter-panel
    HTMX partial. Returns render_filter_panel(state) as HTML.

  GET /healthz
    Returns a minimal health check response (JSON or text).


QUERY PARAMETER NAMES (identical in every language)

  team_in, team_out, status_in, status_out, region_in, region_out

Multi-value: ?status_out=archived&status_out=paused produces status_out = ["archived","paused"].


RENDERING CONTRACT (identical HTML structure in every language)

The filter panel HTML must contain the following structure. Any language-specific
formatting of attributes is acceptable as long as the attribute names and values
match exactly.

Quick excludes strip (at the top of the panel):
  For each (dimension, value) in QUICK_EXCLUDES:
    <button
      data-role="quick-exclude"
      data-quick-exclude-dimension="{dimension}"
      data-quick-exclude-value="{value}"
      data-active="{true|false}"
      ...
    >
      {value} <span data-role="quick-exclude-count">{exclude_impact_counts[value]}</span>
    </button>
  Wrap in: <div data-role="quick-exclude-group">...</div>

Per dimension group (one per facet dimension):
  <section data-filter-dimension="{dimension}">
    <h3>{dimension heading}</h3>

    Include sub-section:
      <div data-role="include-options">
        For each option in FACET_OPTIONS[dimension]:
          <label
            data-filter-dimension="{dimension}"
            data-filter-option="{option}"
            data-filter-mode="include"
            data-option-count="{count}"
            [data-excluded="true" if option is in D_out]
            class="... [opacity-50 cursor-not-allowed if excluded] ..."
          >
            <input type="checkbox" name="{dimension}_in" value="{option}"
              [checked if option is in D_in]
              [disabled if option is in D_out] />
            {option label}
            <span data-role="option-count">{count}</span>
          </label>
          Where count = 0 if option is in D_out, else facet_counts[option].
      </div>

    Exclude sub-section:
      <div data-role="exclude-options">
        For each option in FACET_OPTIONS[dimension]:
          <label
            data-filter-dimension="{dimension}"
            data-filter-option="{option}"
            data-filter-mode="exclude"
            data-option-count="{exclude_impact_counts[option]}"
            [data-active="true" if option is in D_out]
          >
            <input type="checkbox" name="{dimension}_out" value="{option}"
              [checked if option is in D_out] />
            {option label}
            <span data-role="option-count">{exclude_impact_counts[option]}</span>
          </label>
      </div>
  </section>

HTMX form wrapper (on the full page):
  <form id="report-filters"
    hx-get="/ui/reports/results"
    hx-target="#report-results"
    hx-trigger="change, submit">
    ...filter panel...
    ...results section...
  </form>

Result count badge (OOB in partials, inline on full page):
  <div id="result-count"
    [hx-swap-oob="true" in partials only]
    data-role="result-count"
    data-result-count="{N}">
    {N} results
  </div>

Results section:
  <section id="report-results"
    data-query-fingerprint="{fingerprint}"
    data-result-count="{N}">
    <div data-role="active-filters">...</div>
    ...result cards...
  </section>

Add a new doc:

  examples/canonical-api/README-faceted-filter-examples.md

This document must:
- List all language implementations of the faceted filter example.
- State the shared dataset and expected counts (so implementors can self-check):
    With no filters active:
      result_count = 6
      facet_counts[team][growth] = 3, facet_counts[team][platform] = 3
      facet_counts[status][active] = 4, facet_counts[status][paused] = 1,
      facet_counts[status][archived] = 1
    With status_out=archived:
      result_count = 5
      facet_counts[status][active] = 4, facet_counts[status][paused] = 1
      exclude_impact_counts[status][archived] = 1
      include option for "archived" → count = 0, greyed out
    With team_in=growth AND status_out=archived:
      result_count = 3
      exclude_impact_counts[status][archived] = 0
        (no archived rows survive the team_in=growth filter, so archived removes nothing)
      facet_counts[team][growth] = 3, facet_counts[team][platform] = 0
        (platform is filtered out by team_in=growth on other dimensions)
      Wait — recalculate:
        team_in=growth means facet_counts[status] is computed with team_in=growth applied.
        So facet_counts[status][active] = 3 (growth+active), paused = 0, archived = 0.
        exclude_impact_counts[status] is computed WITHOUT any status filters.
        With team_in=growth applied: growth has active(3), archived(0), paused(0).
        So exclude_impact_counts[status][archived] = 0 (no growth+archived rows).
        The archived include option is greyed with count=0 (archived is in status_out).
        The archived exclude option shows count=0 (removing archived removes 0 rows
        from the growth-filtered set, since no growth rows are archived in this dataset).
        This edge case should be documented: exclude impact count CAN be 0 when the

SECTION 7 — Important constraints

- Do NOT re-invent the filter logic. Every language implements the same algorithm.
  The only difference is syntax and rendering idioms. If two languages use the
  same algorithmic approach (e.g., both use sorted list comparison), the code
  can look similar — that is expected and correct.

- Do NOT add pagination, search, or Plotly chart integration to these examples.
  Keep each example focused exclusively on the split include/exclude filter panel.
  The Python example includes Plotly; the non-Python examples do not need it.

- Do NOT use external template files unless the framework requires them (Go templ,
  Zig Jetzig/Zap). For all other languages, build HTML strings inline.

- Do NOT introduce new dependencies beyond what each stack already uses.
  Each stack context doc specifies the canonical dependency set.

- Keep the dataset constant. Do not generate it dynamically. Inline the six rows
  as a literal constant in every language.

- The companion .md file is required for every language. Do not skip it.

- All HTML output must contain valid data-* attributes as specified in the
  RENDERING CONTRACT above. These are the test hooks. Missing attributes will
  cause Playwright assertions to fail.

- When in doubt about a language-specific idiom (e.g., how to add a custom
  data attribute in TyXML, or how to iterate in a templ component), consult
  the existing html-fragment-example for that language and follow its style.
