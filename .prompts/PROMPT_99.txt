BACKGROUND: WHY TOGGLEABILITY
================================================================================

The three new capabilities added in PROMPT_96–98 are powerful but not universally
appropriate:

  - `budget-report` is always available but derived repos may not want it surfaced
    by default in their session context
  - `startup-trace write` is useful for large sessions but adds ceremony to small ones
  - `route-check` is useful for complex repos with many stacks but unnecessary for
    simple single-purpose repos

Derived repos should be able to:
  1. Disable all three capabilities (quiet mode — the default for simple repos)
  2. Enable them selectively per capability
  3. Have the Session Context Briefing reflect which capabilities are available

The toggleability mechanism must:
  - Be explicit and inspectable (in a version-controlled config file)
  - Propagate cleanly from the base repo's profile-rules.json through acb_payload.py
    into the generated repo's .acb/profile/selection.json
  - Not break any existing behavior when disabled

================================================================================
PART 1: ADD startup_features TO profile-rules.json
================================================================================

In `context/acb/profile-rules.json`, add a new top-level key:

  "startup_features": {
    "budget_report_enabled": false,
    "startup_trace_enabled": false,
    "route_check_enabled": false
  }

Rationale for defaults being false:
  - Generated repos should be quiet by default
  - The features are opt-in for repos that want context validation
  - `agent-context-base` itself is a meta-repo where these features are expected;
    simple derived repos should not have them thrust upon them

Note: `agent-context-base` should behave as if all three are enabled (since the commands
exist). The defaults in profile-rules.json are for what gets propagated to derived repos.
A derived repo operator can set them to true in their local profile-rules.json before
running acb_payload.py to regenerate.
