SHARED DOMAIN: TaskFlow Monitor
  A job/task queue monitor. Loads jobs from a fixture file. CLI surface for scripting.
  TUI surface for interactive inspection. Shared domain core used by both surfaces.

JOB SCHEMA:
  {
    "id": "job-001",
    "name": "build-frontend",
    "status": "done",          // pending | running | done | failed
    "started_at": "2024-01-15T10:23:00Z",  // null if pending
    "duration_s": 47.3,        // null if not yet completed
    "tags": ["frontend", "ci"],
    "output_lines": ["Step 1/5: npm install", "Step 2/5: npm run build", "Build complete."]
  }

EVENT SCHEMA:
  {
    "event_id": "evt-001",
    "job_id": "job-001",
    "event_type": "job_started",  // job_queued | job_started | job_completed | job_failed
    "timestamp": "2024-01-15T10:23:00Z",
    "message": "Build started on worker-1"
  }

FIXTURE PATH: examples/canonical-terminal/fixtures/

CLI COMMANDS (shared across all languages):
  taskflow list [--status STATUS] [--tag TAG] [--output json|table]
  taskflow inspect JOB_ID [--output json|plain]
  taskflow stats [--output json|plain]
  taskflow watch [--interval SECS] [--no-tui]   # dual-mode: --no-tui for headless

--- README.md ---

  # TaskFlow Monitor — Python (Typer + Textual)

  Canonical Python dual-mode terminal example. Monitors a fixture-backed job queue
  with a CLI surface (Typer) and TUI surface (Textual).

  ## Stack
  - CLI: Typer
  - TUI: Textual
  - Python 3.11+

  ## Usage
  ```bash
  # Install
  pip install -e .

  # CLI
  taskflow list
  taskflow list --status failed --output json
  taskflow inspect job-001
  taskflow stats

  # TUI (interactive)
  taskflow watch

  # Headless / CI
  taskflow watch --no-tui
  ```

  ## Architecture
  - core/: domain logic (models, loading, filtering, stats) — no I/O
  - cli/:  Typer commands
  - tui/:  Textual app and widgets
  - tests/smoke/: fixture-backed CLI smoke tests (no TTY required)

  ## Testing
  ```bash
  pytest tests/smoke/           # CLI smoke tests (CI-safe)
  pytest tests/unit/            # domain core unit tests
  ```

  ## When to Use This Stack
  Python + dual-mode tooling, operator consoles, interactive job monitors,
  or any tool needing both scriptable CLI and a rich full-screen TUI.

  # TaskFlow Monitor — Python (Click + Blessed)

  Secondary Python terminal example demonstrating a lighter interactive approach
  using Click for CLI and Blessed for colored/interactive terminal output.

  ## Stack
  - CLI: Click
  - Terminal UI: Blessed (lightweight, not a full-screen TUI framework)

  Lighter Python CLIs where a full-screen TUI is overkill. Blessed shines for
