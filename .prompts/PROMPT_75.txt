IV. CLI CHANGES

In build_parser():

  parser.add_argument(
      "--use-example",
      type=int,
      metavar="N",
      help="Pre-fill archetype, stack, and flags from reference project N (1-100). "
           "repo_name defaults to NNN-codename unless supplied explicitly.",
  )
  parser.add_argument(
      "--list-examples",
      action="store_true",
      help="Print all 100 reference project examples and exit.",
  )

In main():
  - Before the existing --list-* checks, handle --list-examples (print and return 0).
  - After parsing args, if --use-example is set:
      - Look up the ExampleProject; error if not found.
      - Populate args.archetype, args.primary_stack if not already set by the caller.
      - Set args.dokku, args.smoke_tests, args.integration_tests, args.seed_data from the
        entry only if the caller did not explicitly pass those flags.
      - If args.repo_name is None (not provided), set it to f"{N:03d}-{entry.codename}".
  - The validation checks (repo_name required, archetype required, primary_stack required)
    run after this resolution, so a bare --use-example 47 is sufficient.

Note: argparse boolean flags default to False, so there is no built-in way to distinguish
"caller explicitly passed --smoke-tests" from "defaulted to False". The simplest correct
approach: apply the example's flag values as defaults, then let the existing parse_args
output override them only if the caller explicitly included the flag. Since argparse
store_true flags are always False unless the flag appears, simply do:
  if not args.smoke_tests:
      args.smoke_tests = entry.smoke_tests
  (and so on for each flag)

This means: if the example has smoke_tests=True, --use-example pre-sets it; the user can
suppress it by explicitly wiring up a --no-smoke-tests pattern if they want. For now,
one-way defaulting (example sets True, caller can only add more, not subtract) is the
correct minimal behavior.
