# SECTION 5 â€” Subagent Roles

`AGENT_MAIN`

- Allowed actions: choose execution mode, assign the active subagent, run the prompt loop, coordinate commits, stop on drift
- Forbidden actions: skipping artifact gates, skipping auditor review, batching multiple prompts into one commit set
- Required outputs: prompt announcement, execution summary, next-prompt decision
- Stop conditions: any gate failure, dependency mismatch, missing target path for non-deterministic prompts

`AGENT_ARCHITECT`

- Allowed actions: interpret architecture prompts, map dependencies, write design/control documents, declare output targets for doc-heavy prompts
- Forbidden actions: inventing new tracks, changing naming conventions, silently relaxing gates
- Required outputs: dependency-aware design artifact, explicit artifact map, rationale for any inferred target files
- Stop conditions: contradictory prompt requirements, unresolved artifact target, unsupported mixed suffix interpretation

`AGENT_BUILDER`

- Allowed actions: create or modify repo files, implement scripts/examples/templates/tests, wire runtime/tooling artifacts
- Forbidden actions: overwriting templates in place, committing without tests when tests are applicable, renaming prompt files automatically
- Required outputs: changed files, runnable artifacts, smoke/integration tests where applicable
- Stop conditions: missing tools, failing tests, Docker inconsistency

`AGENT_AUDITOR`

- Allowed actions: verify artifacts, inspect diffs, confirm tests, validate commit prefixes and scope
- Forbidden actions: ignoring legacy-history ambiguity, marking prompts complete from commits alone
- Required outputs: pass/fail gate decision, drift notes, test verdict, completion status
- Stop conditions: gate failure, malformed commit prefix, incoherent artifact evidence

`AGENT_REFACTOR`

- Allowed actions: resolve drift between prompt intent and repo state, tighten existing outputs, improve coherence after a completed stage
- Forbidden actions: changing prompt numbering, expanding scope into later prompts, retroactively declaring completion without evidence
- Required outputs: scoped corrective diff plus updated gate results
- Stop conditions: refactor would cross prompt boundaries or invalidate prior accepted artifacts
