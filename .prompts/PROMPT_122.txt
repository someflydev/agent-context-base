================================================================================
BACKGROUND
================================================================================

JavaScript/TypeScript and Go represent two very different generation idioms:
  - JS/TS: dynamic language, flexible structures, the challenge is correctness
    under type-optional-ness; @faker-js/faker is comprehensive, Chance adds
    probability helpers and repeatable custom distributions
  - Go: statically typed, struct-first; gofakeit populates structs imperatively,
    go-faker/faker uses struct tags for annotation-driven population; the
    contrast between these two styles is the key teaching surface for Go

For both languages the same TenantCore domain is generated and the same
relational integrity rules apply. Each language must produce JSONL output
that can be validated by examples/canonical-faker/domain/validate_output.py.

A-4. src/distributions.ts

This module is the key Chance demonstration. Use the `chance` library for:

  1. Weighted plan selection:
     const PLAN_WEIGHTS = { free: 50, pro: 35, enterprise: 15 }
     export function pickPlan(c: Chance.Chance): string {
         return c.weighted(["free", "pro", "enterprise"], [50, 35, 15])
     }

  2. Weighted region selection (similar pattern).

  3. Poisson-distributed member count per org:
     export function memberCount(c: Chance.Chance): number {
         // Most orgs have 3–10 members. Use Poisson(lambda=4) + 3.
         const count = c.rpg("1d1", { sum: true }) // placeholder; implement properly
         // Actually: sample from a right-skewed distribution, clamp to 3–50
         ...
     }
     Document that Chance provides rpg(), normal(), and custom probability tools.

  4. Edge-case injection:
     export function withEdgeCase<T>(c: Chance.Chance, rate: number, edgeFn: () => T, normalFn: () => T): T {
         return c.bool({ likelihood: rate * 100 }) ? edgeFn() : normalFn()
     }
     Use this in pipeline.ts to inject ~5% edge-case users.
