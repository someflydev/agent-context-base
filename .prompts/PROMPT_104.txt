--- src/core/models.ts ---

  export type JobStatus = 'pending' | 'running' | 'done' | 'failed';

  export interface Job {
    id: string;
    name: string;
    status: JobStatus;
    started_at: string | null;
    duration_s: number | null;
    tags: string[];
    output_lines: string[];
  }

  export interface Stats {
    total: number;
    by_status: Record<JobStatus, number>;
    avg_duration_s: number | null;
    failure_rate: number;
  }

--- src/cli/index.ts ---

  Commander program with:
    list: options --status, --tag, --output [json|table]
    inspect: argument JOB_ID, option --output [json|plain]
    stats: option --output [json|plain]
    watch: options --interval, --no-tui

  All commands accept --fixtures-path or read TASKFLOW_FIXTURES_PATH env var.
  Markers: wrap table output in ## BEGIN_JOBS ## etc.
  watch --no-tui: print current job list once with markers, exit 0.
  watch (default): launch Ink TUI via renderApp().

--- src/tui/App.tsx ---

  Ink App component:
    - Uses useApp() for quit control
    - Keyboard input via useInput() or ink-select-input
    - Layout: Header (stats bar), JobTable (scrollable list), JobDetail (side panel)
    - Key bindings: q=quit, ↑↓=navigate, f=filter, r=refresh
    - Color by status: green=done, red=failed, yellow=running, dim=pending

  Use ink's <Box>, <Text>, <Static> components.
  Render via renderApp() function called from CLI watch command.

--- tests/smoke/cli.smoke.test.ts ---

  Use vitest (preferred) or jest. Use execa or child_process.execSync.

  Tests:
  1. list_table: run binary, assert exit 0, assert markers in stdout
  2. list_json: --output json, parse JSON, assert length 20
  3. filter_status: --status done --output json, assert all done
  4. inspect_job: inspect job-001 --output json, assert id
  5. stats_json: stats --output json, assert total 20
  6. watch_no_tui: watch --no-tui, exit 0, markers present
  7. missing_fixtures: bad --fixtures-path, exit 1

PART 4: JAVA SECONDARY — JCommander + JLine (taskflow-java-jline)
================================================================================

Create: examples/canonical-terminal/java-jcommander-jline/

Tech stack: JCommander for CLI, JLine 3 for interactive shell-like REPL.

--- Scope ---

This example demonstrates a different interaction model: instead of a full-screen
TUI, the tool provides a shell-like REPL with readline editing, tab completion,
and command history. The `watch` command enters a read-eval loop rather than a
full-screen dashboard.

JLine capabilities to demonstrate:
  - LineReader with history (PersistentHistory or MemoryHistory)
  - Tab completion (Completer for job IDs and status values)
  - ANSI color output via AttributedString / AttributedStyle
  - Terminal.writer() for formatted output

JCommander capabilities: @Parameters for positional args, @Parameter for options.
For non-interactive mode: run commands from args (--no-repl), exit immediately.

Interactive mode (watch without --no-repl):
  Prompt: taskflow>
  Commands typed at prompt: list, inspect <id>, stats, filter status:failed, quit
  Tab completion: job IDs for inspect, status values for filter

--- pom.xml ---

  JCommander 1.82, JLine 3.25, Jackson 2.17, JUnit 5.10.
  Fat JAR (maven-shade).
