--- lib/taskflow/interactive/inspector.rb ---

  class Inspector
    def initialize(fixtures_path)
      @fixtures_path = fixtures_path
      @prompt = TTY::Prompt.new
    end

    def run
      loop do
        jobs = Core::Loader.load_jobs(@fixtures_path)
        action = @prompt.select("TaskFlow Inspector", [
          "List all jobs",
          "Filter by status",
          "Inspect a job",
          "Show stats",
          "Quit"
        ])
        case action
        when "List all jobs"
          display_jobs(jobs)
        when "Filter by status"
          status = @prompt.select("Select status:", %w[pending running done failed])
          filtered = Core::Filter.filter(jobs, status: status)
          display_jobs(filtered)
        when "Inspect a job"
          job_id = @prompt.ask("Enter job ID (e.g. job-001):")
          job = jobs.find { |j| j.id == job_id }
          job ? display_job(job) : @prompt.warn("Job not found: #{job_id}")
        when "Show stats"
          display_stats(Core::Stats.compute(jobs))
        when "Quit"
          break
        end
        @prompt.keypress("Press any key to continue...")
      end
    end
  end

This demonstrates TTY::Prompt's strength: guided, beautiful, wizard-like workflows.
It is NOT a ncurses-style full-screen TUI. That is the honest Ruby answer.

  Flagship Ruby terminal example. Guided interactive CLI inspector using
  TTY::Prompt menus and selections. CLI structure via Thor.

  ## Stack: Thor + TTY::Prompt + TTY::Table

  ## Usage
  ```bash
  bundle install
  ```

  ```bash
  bundle exec ruby tests/smoke/test_cli_smoke.rb
  ```

  ## Ruby Terminal Philosophy
  Ruby's terminal sweet spot is guided interactive workflows — beautiful prompts,
  select menus, confirmation dialogs. This example deliberately avoids forcing
  a full-screen ncurses-style TUI, which is not Ruby's natural fit.

  For dashboard-style full-screen TUI panels in Ruby, see external gems like
  curses. For operator-quality terminal dashboards, prefer the Rust/Go/Python
  examples in this collection.

  ## When to Use
  Operator tools, guided remediation workflows, interactive installers,
  configuration wizards, triage tools — wherever a prompt-driven UX is the goal.

BEAM GUIDANCE:

  # TaskFlow Monitor — Elixir (Optimus + Ratatouille)

  Flagship Elixir dual-mode terminal example. Optimus for CLI parsing.
  Ratatouille for a declarative TUI. JobStore GenServer for state management.

  ## Stack: Optimus + Ratatouille + GenServer

  ## Usage
  ```bash
  mix deps.get
  mix escript.build
  ./taskflow list
  ./taskflow list --status failed --output json
  ./taskflow watch --no-tui
  ./taskflow watch   # launches Ratatouille TUI
  ```

  ## Testing
  ```bash
  mix test
  ```

  ## BEAM Architecture
  JobStore is a GenServer holding job state. This is justified because it cleanly
  models the BEAM pattern for live-data terminal apps: TUI sends GenServer messages;
  the GenServer handles refresh and state updates independently of rendering.
  For a purely static display, the GenServer could be eliminated.

  ## When to Use
  Elixir/BEAM shops needing a terminal operator console where OTP supervision,
  process isolation, and BEAM concurrency are genuine assets.
