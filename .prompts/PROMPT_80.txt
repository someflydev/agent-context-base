IV. TARGET RESOLUTION RULES
================================================================================

Add explicit target-resolution helpers in `scripts/new_repo.py`.

For derived generation, apply these rules:

1. If the selector is a leaf derived example and `--target-dir` is omitted:
   use `Path.cwd() / selector_name`.

2. If `--target-dir` points to an existing directory:
   treat it as a parent directory and resolve to:
     parent / selector_name
   for a leaf selector, or:
     parent / child_name
   for each team-selected child.

3. If `--target-dir` does not exist:
   for a leaf selector, treat it as the exact repo path requested by the user.
   for a team selector, require that the path either already exists as a
   directory or is creatable as a parent directory; then create children under it.

4. Handle `/tmp` and `/private/tmp` equivalently on macOS once resolved.

5. For team selectors, precompute all child targets and validate them all before
   writing anything. If one child would fail because the directory exists and is
   non-empty without `--force`, exit non-zero before writing any child repo.
