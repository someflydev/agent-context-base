BACKGROUND: RUST TEACHING SURFACE
================================================================================

The Rust implementation teaches:

  - plotly.rs typed builders: Plot, Scatter, Bar, Histogram, BoxPlot,
    HeatMap, Funnel — each with typed trace and layout methods
  - serde + serde_json for figure serialization (plotly.rs implements
    Serialize on all its types)
  - Askama compile-time templates: define struct templates with typed fields;
    templates are verified at compile time (no runtime rendering errors)
  - Axum typed extractors: use Query<FilterState> for filter parsing
  - tokio async runtime throughout
  - Separation of domain logic, aggregation, and figure building with
    Rust's module system

Key Rust idiom: plotly.rs's Plot type accumulates traces and returns
plot.to_json() as a String. In handlers, do NOT call to_json() inside the
figure builder. Instead, return the Plot (or an equivalent struct wrapping
it) and serialize in the handler — matching the three-layer model.

If plotly.rs's Plot type cannot be returned from a figure builder without
serialization (e.g., it does not implement Clone), design a FigureData newtype
or serialize to serde_json::Value inside the builder and annotate with a
comment explaining why this deviates from the three-layer model.

PART 5: FIGURE BUILDER LAYER (src/charts/)
================================================================================

Implement all 7 figure builders using the plotly.rs crate.

Each module exposes:
  pub fn build_<name>_figure(agg: &<ResultType>) -> serde_json::Value

Return serde_json::Value (not Plot, not String) from figure builders. This
keeps builders testable: tests can assert on JSON structure without running
a server. Serialize with serde_json::to_value(plot)? where plot is a
plotly::Plot or plotly::common::Plot.

This is a deliberate deviation from the three-layer model's "return structured
object, not JSON string" guidance — returning serde_json::Value IS the
structured object here. Document this in a module-level comment in src/charts/mod.rs:

  // Rust figure builders return serde_json::Value rather than a typed struct
  // because plotly.rs's Plot type does not implement Clone or meaningful
  // introspection. serde_json::Value is the structured form that handlers
  // serialize to a response body, and test code can assert on JSON structure.
  // This matches the spirit of the three-layer model: aggregation → builder →
  // handler, with the builder returning a testable object.

Chart type specifics per family (match Python and Go implementations):

  src/charts/time_series.rs:
    use plotly::Scatter;
    One Scatter trace per environment. Mode::LinesMarkers.
    HoverTemplate set on each trace.

  src/charts/service_bar.rs:
    use plotly::Bar;

  src/charts/latency_histogram.rs:
    use plotly::Histogram;
    X values = agg.values. Annotate P50 as a vertical line via layout shapes.

  src/charts/latency_boxplot.rs:
    use plotly::BoxPlot;
    One trace per service. BoxPoints::Outliers.

  src/charts/incident_bar.rs:
    use plotly::Bar;
    Ordered by severity level (critical > high > medium > low).

  src/charts/funnel_chart.rs:

    Document the substitution clearly.

  src/charts/heatmap_chart.rs:

All builders must add a "meta" key to the returned serde_json::Value with
total_count, filters_applied, and generated_at. Add it via:

    "total_count": total_count,

Empty-state: when aggregate data is empty, return a minimal valid figure with
