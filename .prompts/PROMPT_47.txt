SECTION 2 — Phoenix mini-app file structure

Create `examples/canonical-api/phoenix-example/` with the following files.
Implement each file fully.

**`mix.exs`**

Standard Mix project. App: `:example`, version `0.1.0`, elixir `~> 1.16`.
Application mod: `{Example.Application, []}`.
Dependencies:
- `{:phoenix, "~> 1.7"}`
- `{:bandit, "~> 1.2"}`
- `{:jason, "~> 1.4"}`

**`config/config.exs`**

Configure `ExampleWeb.Endpoint` for the `:example` app:
- `adapter: Bandit.PhoenixAdapter`
- `http: [ip: {0, 0, 0, 0}, port: 4000]`
- `server: true`
Also set `config :phoenix, :json_library, Jason`.

**`config/runtime.exs`**

Read `PORT` from the environment (default `"4000"`) and override the endpoint port:
```elixir
port = String.to_integer(System.get_env("PORT", "4000"))
config :example, ExampleWeb.Endpoint, http: [ip: {0, 0, 0, 0}, port: port]
```

**`lib/example/application.ex`**

OTP Application that starts `ExampleWeb.Endpoint` as the only supervised child.

**`lib/example_web.ex`**

The `ExampleWeb` macro helper module. Define a `controller/0` function that
quotes `use Phoenix.Controller, formats: [:html, :json]` and `import Plug.Conn`.
Define `__using__(which)` to dispatch to the function.

**`lib/example_web/endpoint.ex`**

Minimal `Phoenix.Endpoint` for `:example`. Plug chain:
1. `Plug.RequestId`
2. `Plug.Logger, log: :debug`
3. `ExampleWeb.Router`

**`lib/example_web/router.ex`**

Two scopes:
- `scope "/", ExampleWeb` piped through `:api` (`plug :accepts, ["json"]`):
  - `GET /healthz → HealthController, :show`
  - `GET /api/reports/:tenant_id → ReportController, :index`
  - `GET /data/chart/:metric → ChartController, :show`
- `scope "/fragments", ExampleWeb` piped through `:html` (`plug :accepts, ["html"]`):
  - `GET /report-card/:tenant_id → FragmentController, :show`

**`lib/example/reporting.ex`**

The context module `Example.Reporting`. Define:
- `list_recent_reports(tenant_id, opts \\ [])` — returns a list of maps with
  `%{report_id: ..., total_events: ..., status: ...}`. Seed data: acme has
  `daily-signups` (42 events, `"ready"`), globex has `ops-latency` (17 events,
  `"warming"`). Unknown tenants get the acme row as a fallback.
- `series_for(metric)` — returns a list of `%{x: ..., y: ...}` points.
  Seed data: `signups` metric has three points: `{"2026-03-01", 18}`,
  `{"2026-03-02", 24}`, `{"2026-03-03", 31}`. Unknown metrics get these same
  points as fallback.

The `opts` keyword list supports `limit:` (default 20). Apply `Enum.take/2`.

**`lib/example_web/controllers/health_controller.ex`**

```elixir
defmodule ExampleWeb.HealthController do
  use ExampleWeb, :controller

  def show(conn, _params) do
    json(conn, %{status: "ok", service: "phoenix-example"})
  end
end
```

**`lib/example_web/controllers/report_controller.ex`**

`use ExampleWeb, :controller`. `index/2` extracts `tenant_id` from params,
reads `limit` with `Map.get(params, "limit", "20") |> String.to_integer() |> min(100)`,
delegates to `Example.Reporting.list_recent_reports/2`, returns JSON:
```json
{"service": "phoenix-example", "tenant_id": "...", "reports": [...]}
```

**`lib/example_web/controllers/fragment_controller.ex`**

`use ExampleWeb, :controller`. `show/2` extracts `tenant_id`, calls
`Example.Reporting.list_recent_reports(tenant_id, limit: 1)`, takes the first
report (or a fallback), and returns HTML via `html/2`:

```html
<section id="report-card-{tenant_id}" class="report-card" hx-swap-oob="true">
  <strong>Tenant {tenant_id}</strong>
  <span>{total_events} events in the last hour</span>
  <small>Status: {status}</small>
</section>
```

Use a private helper `render_fragment/3` that builds this string.

**`lib/example_web/controllers/chart_controller.ex`**

`use ExampleWeb, :controller`. `show/2` extracts `metric`, calls
`Example.Reporting.series_for/1`, returns JSON:
```json
{"metric": "...", "series": [{"name": "...", "points": [{"x": "...", "y": 0}]}]}
```

**`Dockerfile`**

```dockerfile
FROM elixir:1.17-alpine

WORKDIR /app

RUN mix local.hex --force && mix local.rebar --force

COPY mix.exs .
RUN mix deps.get

COPY config ./config
COPY lib ./lib

EXPOSE 4000
CMD ["mix", "phx.server"]
```

**`verification/scenarios/phoenix_min_app/verify.py`**

Follow the pattern of `verification/scenarios/rust_axum_min_app/verify.py` exactly:

- `RUNTIME_DIR = REPO_ROOT / "examples/canonical-api/phoenix-example"`

- `native_smoke_check()`:
  - Skip if `mix` is not available (use `command_available("mix")`)
  - Run `["mix", "deps.get"]` with timeout 120 seconds
  - Run `["mix", "compile"]` with timeout 180 seconds
  - Assert both returncode == 0

- `docker_smoke_check()`:
  - Skip if `VERIFY_DOCKER != "1"` or docker is not available
  - Build image with `["docker", "build", "-t", tag, "."]`, timeout 600 seconds
  - Run container: `["docker", "run", "--rm", "-d", "-p", f"{port}:4000", tag]`
  - Call `wait_for_http_json(f"http://127.0.0.1:{port}/healthz", timeout=35.0)`
  - Assert status 200 and `payload["status"] == "ok"`
  - Clean up container in `finally`
