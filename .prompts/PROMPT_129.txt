PART 4: AGGREGATION LAYER (analytics/)
================================================================================

Implement all six aggregation modules following the three-layer model from
context/skills/plotly-figure-builder-design.md.

Each module must expose one or more functions with this signature pattern:
  aggregate_<name>(data: list[<Model>], filters: FilterState) → <AggregateResult>

Where <AggregateResult> is a typed dataclass or NamedTuple. Never return raw
dicts from aggregation functions — use typed results.

Guidelines per module:

  analytics/events.py
    aggregate_event_counts(events, filters) → EventCountSeries
      Returns: dates: list[str], counts: list[int], by_environment: dict[str, list]

  analytics/services.py
    aggregate_service_error_rates(events, services, filters) → ServiceErrorRates
      Returns: services: list[str], error_rates: list[float], total_events: list[int]

  analytics/distributions.py
    aggregate_latency_histogram(samples, filters) → LatencyHistogram
      Returns: values: list[float], bin_size: float, p50: float, p95: float, p99: float
    aggregate_latency_by_service(samples, services, filters) → LatencyByService
      Returns: services: list[str], p50s: list[float], p95s: list[float], p99s: list[float]

  analytics/heatmap.py
    aggregate_event_heatmap(events, filters) → EventHeatmap
      Returns: hours: list[int], days: list[str], counts: list[list[int]]

  analytics/funnel.py
    aggregate_funnel_stages(sessions, filters) → FunnelStages
      Returns: stages: list[str], counts: list[int], drop_off_rates: list[float]

  analytics/incidents.py
    aggregate_incident_severity(incidents, filters) → IncidentSeverity
      Returns: severities: list[str], counts: list[int], mttr_by_severity: dict[str, float]

ALL aggregation functions must:
  - apply all non-empty fields of FilterState
  - return a valid typed result even when input list is empty (graceful empty state)
  - be deterministic: same input, same output

PART 5: FIGURE BUILDER LAYER (charts/)
================================================================================

Implement all figure builders following the three-layer model exactly.

Each module must expose one primary function:
  build_<name>_figure(aggregate: <AggregateResult>) → dict

The returned dict must be serializable with json.dumps() and consumable by
Plotly.js's Plotly.newPlot(). It is the complete Plotly figure JSON.

Mandatory fields in every returned dict:
  - data: list of traces (each with hovertemplate set, name set)
  - layout.title.text: non-empty string
  - layout.xaxis.title.text: non-empty (where applicable)
  - layout.yaxis.title.text: non-empty (where applicable)
  - meta: {"total_count": int, "filters_applied": list[str], "generated_at": str}
    (the meta key is not rendered by Plotly but is useful for tests and summaries)

charts/time_series.py:
  build_time_series_figure(agg: EventCountSeries) → dict
  Chart type: go.Scatter (mode="lines+markers"), one trace per environment
  Use context/skills/chart-type-selection.md justification: trend over time, 1–5 series

charts/service_bar.py:
  build_service_bar_figure(agg: ServiceErrorRates) → dict
  Chart type: go.Bar(orientation="h"), sorted descending by error rate
  Justification: category comparison, ranked (horizontal sorted bar)

charts/latency_histogram.py:
  build_latency_histogram_figure(agg: LatencyHistogram) → dict
  Chart type: go.Histogram
  Annotate P50 with a vertical line (go.layout.Shape)
  Justification: distribution shape

charts/latency_boxplot.py:
  build_latency_boxplot_figure(agg: LatencyByService) → dict
  Chart type: go.Box, one trace per service
  Show P25, P50, P75, P95, P99 whiskers
  Justification: distribution + summary stats

charts/incident_bar.py:
  build_incident_bar_figure(agg: IncidentSeverity) → dict
  Chart type: go.Bar, ordered by severity level (critical > high > medium > low)
  Justification: category comparison

charts/funnel_chart.py:
  build_funnel_figure(agg: FunnelStages) → dict
  Chart type: go.Funnel
  Include drop-off percentage in hover text
  Justification: process drop-off / funnel

charts/heatmap_chart.py:
  build_heatmap_figure(agg: EventHeatmap) → dict
  Chart type: go.Heatmap
  x-axis: hours (0–23), y-axis: days of week (Mon–Sun)
  Colorscale: "Blues" (sequential, not rainbow)
  Justification: time-of-day pattern
