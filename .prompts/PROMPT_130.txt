  - All 6 chart families

  - All 4 filter parameters (date_from, date_to, services, environment, severity)

  - Reading from smoke.json and small.json

  - Go struct types instead of Python dataclasses
  - encoding/json struct serialization instead of plotly.Figure.to_dict()

  - Go testing package instead of pytest
  - Different directory layout (Go module conventions)
  - Different port (e.g., 8080 vs Python's 8000)

PART 3: PLOTLY FIGURE STRUCTS
================================================================================

Implement internal/charts/plotly.go with explicit Go struct types for the
Plotly figure model. These structs encode the Plotly.js JSON structure:

  type PlotlyTrace struct {
    Type         string          `json:"type"`
    Mode         string          `json:"mode,omitempty"`
    Name         string          `json:"name"`
    X            interface{}     `json:"x,omitempty"`
    Y            interface{}     `json:"y,omitempty"`
    Z            interface{}     `json:"z,omitempty"`
    Orientation  string          `json:"orientation,omitempty"`
    HoverTemplate string         `json:"hovertemplate"`
    Marker       *PlotlyMarker   `json:"marker,omitempty"`
    // Add other fields as needed for specific chart types
  }

  type PlotlyMarker struct {
    Colorscale   interface{}   `json:"colorscale,omitempty"`
    Color        interface{}   `json:"color,omitempty"`
    // ...
  }

  type PlotlyAxis struct {
    Title PlotlyAxisTitle `json:"title"`
  }

  type PlotlyAxisTitle struct {
    Text string `json:"text"`
  }

  type PlotlyTitle struct {
    Text string `json:"text"`
  }

  type PlotlyLayout struct {
    Title  PlotlyTitle  `json:"title"`
    XAxis  PlotlyAxis   `json:"xaxis"`
    YAxis  PlotlyAxis   `json:"yaxis"`
    // Add height, margin, legend, shapes fields as needed
  }

  type PlotlyMeta struct {
    TotalCount     int      `json:"total_count"`
    FiltersApplied []string `json:"filters_applied"`
    GeneratedAt    string   `json:"generated_at"`
  }

  type PlotlyFigure struct {
    Data   []PlotlyTrace  `json:"data"`
    Layout PlotlyLayout   `json:"layout"`
    Meta   PlotlyMeta     `json:"meta"`
  }

  // ToJSON serializes the figure. Used in handlers, not in builders.
  func (f PlotlyFigure) ToJSON() ([]byte, error) {
    return json.Marshal(f)
  }

Extend PlotlyTrace with additional fields as you implement each chart type.
Do not attempt to cover all possible Plotly fields up front — add fields as needed.

PART 5: AGGREGATION LAYER (internal/analytics/)
================================================================================

Implement all six aggregation functions following the three-layer model.

Each function signature pattern:
  func Aggregate<Name>(data []<Model>, filters FilterState) <ResultType>

Types for each result (define in the same file as the aggregation function,
or in a results.go file in the analytics package):

  EventCountSeries: Dates []string, Counts []int, ByEnvironment map[string][]int
  ServiceErrorRates: Services []string, ErrorRates []float64, TotalEvents []int

All aggregation functions must:

  - return a zero-value result (not panic) when the input slice is empty
  - be deterministic: same input yields same output

For percentile calculations (P50, P95, P99 in latency):
  Sort the float64 slice and compute by index. Do not use an external statistics library.
  Include a helper func percentile(values []float64, p float64) float64 in distributions.go.
