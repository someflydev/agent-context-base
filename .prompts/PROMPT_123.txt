BACKGROUND
================================================================================

This prompt covers the typed/JVM cluster plus Rust:

RUST: fake-rs is the primary Rust library. It provides derive macros and a
      typed generation API. The key teaching surface is building a relational
      builder layer on top of fake-rs that maintains ID pools and enforces the
      7-stage generation order. Rust's ownership model makes the pool pattern
      explicit in a way that dynamic languages hide. This is an advantage — show it.
      Use deterministic seeding with fake-rs's Seed trait (or rand::SeedableRng).

JAVA: Datafaker is the modern maintained JVM faker. It supersedes java-faker.
      The Java example's primary teaching value is the orchestration layer above
      Datafaker — showing how to build a relational graph using plain Java data
      structures, not JPA or Spring annotations. The example must be pure Java
      with no framework dependencies.

KOTLIN: Two styles side by side. kotlin-faker provides a DSL-idiomatic Kotlin API.
        Datafaker is available via JVM interop for teams that already use it on
        the Java side. Show both. Use Kotlin data classes, sequence/flow-based
        pipeline idioms, and coroutines if the batched write path benefits from it.
