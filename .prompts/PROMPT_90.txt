--------------------------------------------------------------------------------
WORK TOOL ENHANCEMENT GOALS
--------------------------------------------------------------------------------

Enhance `scripts/work.py` in thoughtful ways that improve practical session management.

Focus especially on these capabilities:

1. Repo-local example-template scaffolding

`init` and `checkpoint` should remain able to scaffold missing runtime files, but the tool should prefer repo-local example templates when they exist.

Desired behavior:

- if repo-local example files exist, use them as the scaffold source for the matching runtime files
- otherwise fall back to the current built-in defaults
- keep this deterministic and simple
- do not require example files to exist

If you choose naming or lookup conventions, make them clear and documented.

2. Better resume/status signal quality

Improve `resume` and `status` so they surface more useful runtime signals such as:

- last commit timestamp or equivalent recent git anchor
- stronger “what changed recently” clues
- better inference of the next concrete step when `TASK.md` already contains one
- clearer indication of whether `PLAN.md` likely deserves review

Do this without pretending to know more than the repo can actually observe.

3. Durable-memory promotion hints

Add grounded heuristics that help assistants notice when a change may deserve promotion into `context/MEMORY.md`, such as when changes touch:

- assistant-facing docs
- templates
- repo-factory or generator behavior
- prompt-sequence or standing workflow doctrine

These should be suggestions, not forced actions.

4. Time-aware staleness signals

The current script already compares runtime files to changed files.

Enhance it with a modest elapsed-time staleness signal as well, so assistants can see when session notes have simply aged enough to deserve review even if the working tree is quiet.

Keep this heuristic modest and explainable.

--------------------------------------------------------------------------------

- `resume` should remain read-only
