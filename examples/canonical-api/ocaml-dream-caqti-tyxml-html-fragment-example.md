# OCaml Dream Caqti TyXML HTML Fragment Example

This example shows the preferred OCaml fragment surface for HTMX-style partial updates:

- keep the Dream route narrow and explicit
- render the fragment through TyXML instead of handwritten HTML assembly
- preserve stable markers such as fragment ids and `hx-swap-oob`
