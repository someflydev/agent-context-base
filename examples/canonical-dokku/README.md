# Canonical Dokku Examples

Use this category for preferred single-service Dokku deployment patterns.

## A Strong Canonical Dokku Example Should Show

- explicit boot path
- clearly documented environment variables
- backing-service expectations
- release or migration behavior if applicable
- post-deploy smoke verification target

## Choosing This Example

Choose this category when deployment support is a first-order concern and the repo is a single deployable service.

## Drift To Avoid

- platform-wide orchestration examples
- hidden release hooks
- examples that treat deployment as proof of persistence correctness
