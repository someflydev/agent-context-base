# Analytics Arc Knowledge

## Overview
The canonical analytics example teaches server-rendered visualization using Plotly, HTMX, and Tailwind across multiple language stacks. It dogfoods the synthetic data generated by the Faker arc to ensure realistic, high-cardinality test data.

## Pattern: Three-Layer Model
1. **Aggregation Layer**: Filters raw fixture data into structured analytical shapes (Time series, Funnels, Heatmaps).
2. **Builder Layer**: Transforms analytical shapes into Plotly-compatible JSON (data, layout, meta).
3. **Presentation Layer**: Renders HTMX-swappable fragments containing `<div id="chart">` and the Plotly script.

## Stack Implementations
- **Python**: FastAPI + Jinja2 + Plotly
- **Go**: Echo + Templ + Plotly
- **Rust**: Axum + Askama + Plotly
- **Elixir**: Phoenix + HEEx + Plotly

## Lessons Learned
- **Fixture Stability**: Deterministic UUIDs and base dates in `generate_fixtures.py` are critical for stable regression tests across stacks.
- **Pre-computed vs. Dynamic**: While some stats are pre-computed in fixtures (error rates), the workbench emphasizes dynamic filtering in memory to simulate high-interactivity UIs without a database.
- **HTMX vs. LiveView**: In the Elixir implementation, HTMX is preferred over LiveView for this specific arc to maintain parity with other stacks and demonstrate the stack-agnostic "backend-driven UI" pattern.
