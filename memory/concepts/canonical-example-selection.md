# Concept: Canonical Example Selection

Use canonical examples as ranked implementation anchors, not as generic samples.

- Start with the active stack, archetype, and manifest.
- Prefer examples already named in manifests or registry metadata.
- Prefer stronger verification levels over weaker ones when relevance is equal.
- Prefer one clean, current example over several near-duplicates.
- Treat README claims as secondary to `verification/example_registry.yaml`.
- Use category READMEs to confirm what a strong example in that family should show.
- Fall back to broader examples only when no tightly matched example exists.

For the full selection rules and ranking workflow, read
`context/skills/canonical-example-selection.md`.
